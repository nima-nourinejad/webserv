Study Plan for Learning and Research Phase

Part 1: Introduction to Web Servers and HTTP

Videos and Articles:

  • "What is a Server? Servers vs Desktops Explained" by PowerCert Animated Videos
  • "How Do Web Servers Work?" by GeeksforGeeks
  • "Web Server Concepts and Examples" by WebConcepts
  • "What are Web Servers and How Do They Work (with examples httpd and nodejs)" by Hussein Nasser

Learning Outcome:

Understand the basics of web servers and the HTTP protocol.

Guided Questions:

  • What is a web server? A web server stores and serves web pages to clients via HTTP.
  • How does the web server communicate with a client? Via HTTP, it responds to browser requests with the requested content.
  • What are the key components of HTTP? Request (method, URL, headers) and response (status code, headers, body).
  • What is the difference between static and dynamic content? Static content is pre-built (HTML, CSS), while dynamic content is generated at request time (using CGI).
  • How do common web servers (like Apache) work? They listen for client requests, process them, and respond using HTTP.

Part 2: Understanding the HTTP Request/Response Cycle

Videos and Articles:

  • "The Http and the Web | Http Explained | Request-Response Cycle" by The TechCave
  • "Understanding HTTP: Basics for Beginners" by DEV Community
  • "How HTTP Protocol Works?" by Animated Study
  • "How the Web Works, HTTP Request/Response Cycle" by Davis Gitonga
  • "Parts of an HTTP Request" by Udacity
  • "HTTP Request-Response Cycle - Tutorial" by unRepo
  • "HTTP Request & Response | ASP.NET Core MVC Course" by procademy

Learning Outcome:

Grasp how requests and responses work in HTTP, including GET, POST, and DELETE methods.

Guided Questions:

  • What is the HTTP request-response cycle? The client sends a request; the server processes it and sends a response.
  • What are the HTTP methods, and when are they used?
    • GET: Retrieve data.
    • POST: Submit data.
    • DELETE: Remove a resource.
  • What are HTTP status codes? Codes that indicate the result of the request:
    • 200: Success
    • 404: Not found
    • 500: Server error
  • What is the purpose of HTTP headers? They provide metadata about the request or response (e.g., content type, user-agent).
  • How does chunked transfer encoding work? Data is sent in chunks for large responses, allowing the server to send it progressively.

Part 3: Deep Dive into C++ and Networking Basics

Videos and Articles:

  • "C++ Programming Tutorial" by freeCodeCamp.org
  • "Computer Network Tutorial" by GeeksforGeeks
  • "Fundamentals of Computer Networking" by Microsoft Learn
  • "Socket Programming in C++ by GeeksforGeeks"
  • "Networking in C++ Part #1: MMO Client/Server, ASIO & Framework Basics by javidx9"
  • "Beej’s Guide to Network Programming by Beej’s Guide"

Learning Outcome:

Gain a solid understanding of C++ programming, socket programming, and basic networking concepts.

Guided Questions:

  • What is socket programming? It enables communication over a network between a client and server using sockets.
  • How do you set up a TCP server in C++? Use socket(), bind(), listen(), accept(), send(), and recv() to handle connections.
  • What is non-blocking I/O? It allows the server to handle multiple clients without waiting for I/O operations to complete.
  • How does a server handle multiple connections? Using event loops (e.g., select(), poll()) or threads to manage clients concurrently.
  • How do you handle errors in socket programming? Use error-checking functions and handle specific errors like timeouts (EAGAIN, ECONNRESET).

Part 4: Review Configuration Files and Server Setup

Videos and Articles:

  • "Virtual Hosting and Apache Server Configuration Basics" by Server Academy
  • "Complete Configuration Example for Nginx" by Nginx
  • "Linux Server Configuration Basics" by DigitalOcean
  • "Apache Web Server Setup and Configuration on Ubuntu" by TechWorld with Nana

Learning Outcome:

Understand how configuration files work in web servers (Nginx, Apache) and basic server setup steps.

Guided Questions:

  • What is a configuration file? A file containing settings for software (like a web server).
  • What are virtual hosts? They allow multiple domains to be hosted on the same server using different configurations.
  • What’s the role of the configuration file in Nginx? It defines server behavior, locations, and how requests are processed.
  • How do you troubleshoot configuration errors? Check logs, test config syntax (nginx -t, apache2ctl configtest), and isolate changes.
  • What are common security practices for web server configurations? Restrict file permissions, use SSL, and configure proper user roles.

Part 5: Exploring CGI and Dynamic Content

Videos and Articles:

  • "What is CGI? How CGI Works?" by Computer Science Hub
  • "CGI Tutorial: How to Use CGI to Generate Dynamic Web Content" by HowToFAQ
  • "Common Gateway Interface (CGI)" by TechTarget

Learning Outcome:

Explore how CGI scripts work to generate dynamic web content.

Guided Questions:

  • What is CGI? A standard that enables web servers to execute scripts and generate dynamic content.
  • How does CGI interact with web servers? The server passes request data to the script and sends the output back as part of the response.
  • What programming languages are commonly used for CGI? Perl, Python, and Bash are commonly used.
  • What is the security concern with CGI? It can execute commands on the server, so improper input handling may lead to vulnerabilities.
  • How does CGI compare to modern alternatives? CGI is slower and less scalable than technologies like FastCGI, mod_perl, or application frameworks.

Tasks to Master by End of Research and Learning Phase

Basic HTTP Protocol (1.1):

  • Handle GET, POST, and DELETE requests.
  • Manage different HTTP response codes (200, 400, 404, 500, etc.).
  • Implement chunked transfer encoding.

Configuration Parsing:

  • Read a configuration file (usually in a .conf format).
  • Define multiple server blocks (virtual hosts).
  • Specify ports, server names, error pages, root directories, and more.

CGI Handling:

  • Handle CGI scripts (e.g., PHP, Python) for dynamic content generation.
  • Support environment variables and interaction with scripts.

Concurrency and Connections:

  • Manage multiple client connections concurrently.
  • Handle both blocking and non-blocking I/O operations (select, poll, or epoll).

Static File Handling:

  • Serve static files such as HTML, CSS, JavaScript, images, etc.
  • Handle file permissions, directories, and indexes (directory listing).

Security and Error Handling:

  • Implement proper error handling and logging.
  • Manage timeouts, request size limits, and other security concerns (e.g., request validation).

Performance:

  • Ensure the server is performant, handling large amounts of requests efficiently.
  • Implement basic load balancing between requests.
